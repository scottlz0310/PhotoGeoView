# 🎬 Qt-Theme-Breadcrumb機能 視覚的デモガイド

## 📋 実際のアプリケーションでの機能確認手順

### 🚀 1. アプリケーション起動

```bash
python main.py
```

アプリケーションが起動すると、以下の新機能が確認できます：

---

## 🎨 2. テーマ管理機能のデモ

### A. ツールバーでのテーマ切り替え
1. **メインウィンドウ上部のツールバーを確認**
   - 「Toggle Theme」ボタンが表示されています
   - クリックするとライト⇄ダークテーマが切り替わります

2. **視覚的変化を確認**
   - 背景色、文字色、ボタンの色が即座に変更
   - 全てのUIコンポーネント（フォルダナビゲーター、サムネイル、ブレッドクラムなど）が統一的に変更

### B. キーボードショートカットでのテーマ切り替え
1. **Ctrl + T を押す**
   - テーマ選択ダイアログが開きます
   - 利用可能なテーマ一覧が表示されます：
     - Default Light (Current) - 現在選択中のテーマには (Current) が表示
     - Dark
     - Photography

2. **テーマを選択して Apply をクリック**
   - 選択したテーマが即座に適用されます
   - ダイアログが自動的に閉じます

3. **Ctrl + Shift + T を押す**
   - テーマが順番に切り替わります
   - Default Light → Dark → Photography → Default Light の順

---

## 🧭 3. ブレッドクラムナビゲーション機能のデモ

### A. ブレッドクラムバーの確認
1. **左パネル最上部を確認**
   - ブレッドクラムバーが表示されています
   - 現在のパス階層が表示されています（例：Home > Documents > Pictures）

2. **パス表示の特徴**
   - 各フォルダがクリック可能なセグメントとして表示
   - 現在のフォルダが右端に表示
   - 長いパスは自動的に省略表示（Home > ... > Pictures > Vacation）

### B. ブレッドクラムでのナビゲーション
1. **セグメントをクリック**
   - 任意のフォルダ名（セグメント）をクリック
   - そのフォルダに即座に移動
   - フォルダナビゲーターとサムネイルグリッドも自動的に更新

2. **キーボードナビゲーション**
   - **Alt + Up**: 親フォルダに移動
   - **Tab**: ブレッドクラムセグメント間を移動（フォーカスが移動）
   - **Enter**: フォーカスされたセグメントに移動

### C. フォルダナビゲーターとの連携確認
1. **フォルダナビゲーターでフォルダを選択**
   - 左パネルのフォルダツリーで任意のフォルダをクリック
   - ブレッドクラムが自動的に更新されることを確認

2. **双方向同期の確認**
   - ブレッドクラムで移動 → フォルダナビゲーターが同期
   - フォルダナビゲーターで移動 → ブレッドクラムが同期
   - サムネイルグリッドも連動して更新

---

## 🔄 4. 統合機能のデモ

### A. テーマとナビゲーションの連携
1. **テーマを変更**
   - Ctrl + T でダークテーマに変更
   - ブレッドクラムバーの色も変更されることを確認

2. **ナビゲーション操作**
   - ダークテーマでブレッドクラムナビゲーションを使用
   - 視認性が向上していることを確認

### B. ファイルシステム監視機能
1. **フォルダを開く**
   - 「File」メニューから「Open Folder」を選択
   - 画像ファイルが含まれるフォルダを選択

2. **自動更新の確認**
   - ブレッドクラムが新しいパスを表示
   - サムネイルグリッドが自動的に更新
   - フォルダナビゲーターも同期

   - ブレッドクラムの文字が見やすく表示

## 📊 6. パフォーマンス確認

### A. テーマ切り替え速度
1. **高速切り替えテスト**
   - Ctrl + Shift + T を連続で押す
   - テーマが瞬時に切り替わることを確認
   - 処理時間: 通常 0.001秒以下

1. **大量ファイルフォルダでのテスト**

   - サムネイル生成と並行して動作

## 🛠️ 7. エラーハンドリングのデモ

### A. 存在しないパスへのアクセス
1. **手動でのテスト**
   - 外部でフォルダを削除
   - アプリケーション内でそのフォルダに移動を試行
   - 適切なエラーメッセージと代替パスへの移動を確認

### B. ネットワークドライブの切断
1. **ネットワークドライブがある場合**
   - ネットワークドライブのフォルダに移動
   - ネットワーク接続を切断
   - 適切なエラーハンドリングを確認

## 💡 9. 使用のコツとベストプラクティス

### 効率的な操作方法
1. **キーボードショートカットの活用**
   - Ctrl + T: 素早いテーマ変更
   - Alt + Up: 階層を上がる
   - Tab + Enter: キーボードのみでナビゲーション

2. **テーマの使い分け**
   - 日中作業: Default Light
   - 夜間作業: Dark
   - 写真編集: Photography

3. **ナビゲーションの効率化**
   - ブレッドクラムで大きく移動
   - フォルダナビゲーターで細かい調整
   - 両方を組み合わせて最適なワークフロー

### トラブルシューティング
1. **動作が重い場合**
   - 大量ファイルフォルダでは処理時間がかかります
   - しばらく待つか、より小さなフォルダに移動

2. **表示が崩れる場合**
   - ウィンドウサイズを調整
   - アプリケーションを再起動
